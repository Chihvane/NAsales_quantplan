# 第三部分策略初稿：中国供应链与出口北美落地分析

## 一、这一部分要回答什么

第三部分不再回答“市场有没有需求”，而是回答另一组更接近落地的问题：

1. 中国供应链能不能稳定供货。
2. 采购成本和到岸成本有没有竞争力。
3. 合规、认证、关税、物流这些门槛是否可控。
4. 哪一种出口与履约路径最适合当前品类。
5. 如果进入北美，这个品类应该如何以最低风险完成第一阶段试运营。

这一部分的输出，不应该只是“供应商很多、可以出口”这种描述性结论，而应该形成一套可执行的落地判断：

- 哪类供应商最值得谈
- 哪种贸易条款更适合
- 哪种物流方式更优
- 哪些合规点必须前置完成
- 单件商品的到岸成本安全边际有多大
- 第一批试单应该如何组织

---

## 二、第三部分在整份报告中的位置

建议把第三部分定义为：

**中国货源、出口路径与到岸成本可行性分析**

它承接前两部分：

- 第一部分告诉我们北美有没有需求、用户在哪里、价格带在哪里。
- 第二部分告诉我们什么商品在卖、卖得好的画像是什么、成交价和竞争结构如何。
- 第三部分则判断：中国供应链是否能以合适的成本、质量、速度和合规方式去承接这个市场机会。

所以第三部分的本质，是把“市场机会”转化为“供应链进入方案”。

---

## 三、第三部分建议的章节结构

建议第三部分固定分成七个章节。

### `3.1 中国供应端结构分析`

这一节回答：国内是否有足够成熟的供给基础。

建议重点分析：

- 主要产业带分布
- 核心工厂类型
- 工厂、贸易商、现货批发商的供给差异
- OEM / ODM 可行性
- 起订量区间
- 打样周期与量产周期
- 供应链成熟度
- 是否存在头部供应商高度集中的情况

这一节的目标不是列工厂名单，而是判断：

- 这个品类在中国是不是成熟供应链品类
- 供应是分散型还是集中型
- 是适合低成本试单，还是需要深度开发

建议输出：

- 供应端成熟度评级
- 产业带分布图
- 供应商类型结构图
- 供应端进入门槛结论

### `3.2 国内采购与报价策略分析`

这一节回答：国内拿货究竟多少钱，价格结构是否清晰。

建议重点分析：

- 工厂出厂价
- 贸易商报价
- 不同 MOQ 下的阶梯价
- 包装、定制、打样、模具、配件的附加成本
- 不同质量档位的价格差
- 不同贸易条款下的报价差异

这一节要特别强调：不能只看单一报价，而要看报价结构。

建议输出：

- 标准 RFQ 询价表
- MOQ 阶梯价曲线
- 不同供应商报价带分布
- 低价、主流、高价三档采购区间
- 建议采购策略

### `3.3 合规、认证与准入门槛分析`

这一节回答：货能不能合法、稳定进入北美并上架销售。

建议重点分析：

- 对应 HS / HTS 编码判断
- 北美目标市场的认证要求
- 安全、环保、无线、电池、发动机等特殊合规要求
- 平台上架所需的材料与测试文件
- 包装、标签、说明书要求
- 知识产权、专利、商标、外观风险

这一节要形成明确区分：

- 必需项
- 建议项
- 风险项
- 暂不确定项

建议输出：

- 合规清单
- 认证路径表
- 合规前置事项表
- 知识产权风险提示表
- 合规复杂度评级

### `3.4 出口路径与物流履约分析`

这一节回答：货从中国到北美，应该怎么走。

建议重点分析：

- EXW / FOB / CIF / DDP 的适用差异
- 海运整柜、海运拼箱、空运、快递的适用品类
- 中国端提货、报关、装运流程
- 美国 / 加拿大清关流程
- 海外仓、FBA、3PL、经销商仓的履约差异
- 时效、成本、风险的平衡关系

这一节的结论不应该只停留在“海运便宜、空运快”，而是要明确：

- 试单期用什么方式
- 稳定补货期用什么方式
- 高客单商品和低客单商品是否使用同一套路径

建议输出：

- 出口流程图
- 物流路径对比表
- 交期结构图
- 履约模式建议

### `3.5 到岸成本与利润安全边际分析`

这一节回答：商品真正落到北美仓后，成本还有没有优势。

建议重点分析：

- 采购成本
- 国内运输
- 报关与出口服务费
- 国际运费
- 保险
- 关税及附加税费
- 目的港费用
- 海外仓 / FBA 入仓费用
- 包装、贴标、售后损耗

第三部分最重要的量化结论，通常会在这一节形成。

建议统一输出三个口径：

- 单件采购成本
- 单件到岸成本
- 单件可售成本

建议输出：

- Landed Cost 拆解表
- 成本瀑布图
- 不同物流方式的单位成本对比
- 成本敏感性分析
- 利润安全边际结论

### `3.6 风险矩阵与应对策略`

这一节回答：如果做这个品类，最容易在哪些地方出问题。

建议重点分析：

- 供应商稳定性风险
- 质量一致性风险
- 合规认证风险
- 关税波动风险
- 清关延误风险
- 运输破损风险
- 汇率波动风险
- 知识产权风险
- 售后返修与召回风险

建议不要只写风险名称，要同步写应对动作。

建议输出：

- 风险热力图
- 高优先级风险清单
- 风险应对动作表
- 是否建议先小批量试单的结论

### `3.7 推荐进入路径与首批执行方案`

这一节回答：如果要做，第一步怎么做。

建议重点分析：

- 先做现货试单还是直接开发定制
- 先做单一 SKU 还是组合 SKU
- 首批订货量建议
- 首选供应商合作模式
- 首选物流和仓储路径
- 首批预算区间
- 第一阶段验证目标

建议输出：

- 首批试运营方案
- 90 天执行路径
- 供应链合作策略建议
- 是否推荐进入的结论

## 四、第三部分的核心判断框架

为了让每个品类都能用同一套逻辑分析，建议第三部分固定围绕以下五个问题做判断：

### 4.1 供给是否成熟

判断标准：

- 是否有成熟产业带
- 是否有足够数量的可替代供应商
- 是否容易拿到稳定报价
- 是否容易组织打样和试单

### 4.2 成本是否可打

判断标准：

- 国内采购价是否能进入目标价格带
- 到岸后是否还有利润空间
- 不同渠道模式下是否仍有成本优势

### 4.3 合规是否可控

判断标准：

- 合规路径是否明确
- 认证成本是否过高
- 是否存在高概率卡关的政策风险

### 4.4 交付是否稳定

判断标准：

- 交期是否稳定
- 是否存在旺季大幅波动
- 供应商是否有持续补货能力

### 4.5 首批试运营是否容易

判断标准：

- 是否可以低 MOQ 启动
- 是否可以小批量验证
- 是否能快速完成上架前准备

建议最终把这五项形成一个固定评分卡。

## 五、第三部分建议固定使用的交付件

为了让这一部分以后能持续复用，建议每份报告都固定输出以下交付件：

1. 供应商类型与产业带地图
2. 供应商报价对比表
3. MOQ 与阶梯价分析表
4. 合规认证清单
5. 出口流程图
6. 物流方式对比表
7. 到岸成本拆解表
8. 成本敏感性分析图
9. 风险矩阵
10. 首批试运营建议表

这样第三部分以后不仅能写成研究报告，也能直接变成执行前的决策文档。

## 六、为后续量化和代码开发预留的标准表结构

第三部分最终也建议像第一、第二部分一样，建立标准输入表。

建议先预留以下几张标准表：

### `suppliers`

用途：记录供应商基础信息与供应能力。

建议字段：

- `supplier_id`
- `supplier_name`
- `supplier_type`
- `region`
- `factory_flag`
- `oem_flag`
- `odm_flag`
- `main_category`
- `moq`
- `sample_days`
- `production_days`
- `capacity_score`
- `compliance_support_score`

### `rfq_quotes`

用途：记录不同供应商、不同 MOQ、不同条款下的报价。

建议字段：

- `supplier_id`
- `sku_version`
- `incoterm`
- `moq_tier`
- `unit_price`
- `sample_fee`
- `tooling_fee`
- `packaging_cost`
- `customization_cost`
- `currency`
- `quote_date`

### `compliance_requirements`

用途：记录品类对应的认证、标签、测试与平台准入要求。

建议字段：

- `market`
- `category`
- `requirement_type`
- `requirement_name`
- `mandatory_flag`
- `estimated_cost`
- `estimated_days`
- `risk_level`
- `notes`

### `logistics_quotes`

用途：记录不同物流方式和目的地的成本与时效。

建议字段：

- `route_id`
- `origin`
- `destination`
- `shipping_mode`
- `incoterm`
- `quote_basis`
- `cost_value`
- `currency`
- `lead_time_days`
- `volatility_score`
- `quote_date`

### `tariff_tax`

用途：记录目标市场对应的关税、附加费、清关费用。

建议字段：

- `market`
- `hs_code`
- `base_duty_rate`
- `additional_duty_rate`
- `brokerage_fee`
- `port_fee`
- `effective_date`

### `shipment_events`

用途：记录实际订单的发货、清关、入仓与上架事件，用于后续回测交期稳定性。

建议字段：

- `shipment_id`
- `supplier_id`
- `shipping_mode`
- `etd`
- `eta`
- `customs_release_date`
- `warehouse_received_date`
- `sellable_date`
- `delay_days`
- `issue_type`

## 七、第三部分建议的量化方向

虽然这一版先写策略初稿，但第三部分后续量化时，建议优先固化以下指标。

### `3.1 供应端成熟度指标`

- 产业带集中度
- 合格供应商数量
- 工厂占比
- 可替代供应商数量
- 平均 MOQ
- 平均打样周期
- 平均量产周期

### `3.2 报价结构指标`

- 均价
- 中位价
- 低价带 / 主流价带 / 高价带
- MOQ 降价斜率
- 工厂与贸易商价差
- 条款价差

### `3.3 合规复杂度指标`

- 必需认证数量
- 预计认证成本
- 预计认证周期
- 平台材料复杂度
- 知识产权风险等级

### `3.4 物流与交付指标`

- 单件国际运费
- 单件总物流成本
- 平均交期
- 交期波动率
- 清关风险评分

### `3.5 到岸成本指标`

- 单件采购成本
- 单件到岸成本
- 单件可售成本
- 成本占零售价比例
- 毛利安全边际

### `3.6 风险指标`

- 高风险项数量
- 高风险项权重分数
- 供应链稳定度评分
- 合规失败风险评分
- 交付中断风险评分

## 八、第三部分建议固定使用的结论模板

为了保证以后不同品类的第三部分可以直接横向比较，建议结论页固定输出以下内容：

- 中国供应链成熟度：高 / 中 / 低
- 成本竞争力：强 / 中 / 弱
- 合规复杂度：高 / 中 / 低
- 物流履约难度：高 / 中 / 低
- 首批试单可行性：高 / 中 / 低
- 推荐进入路径：现货试单 / OEM / ODM / 暂不建议
- 建议首批 MOQ 区间
- 建议首选贸易条款
- 建议首选物流方案
- 关键前置条件

## 九、这一版初稿的定位

这一版是第三部分的策略骨架，不是最终的方法论版本。

它先解决三件事：

1. 明确第三部分在整份报告里的任务边界。
2. 固定第三部分的章节结构与标准交付件。
3. 为下一步量化建模和代码开发预留统一数据接口。

下一步最适合继续往下做的是：

1. 细化第三部分的标准表结构和字段字典。
2. 设计到岸成本模型的统一计算口径。
3. 把合规、关税、物流、采购四条链路拆成独立量化模块。
4. 再进入第三部分代码开发。

---

## 十、建议的下一步工作顺序

如果继续推进，我建议按下面顺序开发第三部分：

1. 先写 `Part 3 Quantitative Structure`
2. 再写 `Landed Cost` 统一公式与参数口径
3. 再设计 `suppliers / rfq_quotes / logistics_quotes / tariff_tax / shipment_events` 五张表
4. 最后再开始第三部分的代码实现与 demo

这样第三部分就会和前两部分一样，既有报告框架，也有可运行的量化底座。
