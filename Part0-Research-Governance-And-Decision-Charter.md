# 第零部分：研究治理与决策总则

## 一、这一部分要回答什么

Part 0 不负责判断某个具体品类是否值得做。

Part 0 负责回答的是另一类更上位的问题：

- 这套研究体系是否具备稳定、可复用、可审计的治理基础
- 任何一个 Part 1–5 的结论是否都能回到统一证据等级和统一 Gate
- 团队在推进、暂停、回滚和退出时，是否有明确的签字链与责任链
- 同一套框架换一个品类、换一个季度、换一个执行人时，结论口径是否还能保持稳定

一句话：Part 0 是整套研究与经营框架的控制总则，而不是普通章节。

## 二、Part 0 在整份报告中的位置

Part 0 位于 Part 1–5 之前。

它不是附录，也不是说明页，而是整个系统的治理层。

建议固定理解为：

- Part 0：治理层与决策层
- Part 1：市场是否值得进入
- Part 2：什么商品画像值得切入
- Part 3：中国供应链是否能稳定承接
- Part 4：先走哪条销售路径
- Part 5：上线后如何监控、扩张、回滚或退出

没有 Part 0，Part 1–5 只能构成一套研究框架。

有了 Part 0，Part 1–5 才能升级为经营控制系统。

## 三、Part 0 建议的章节结构

### `0.1 全局决策树与研究问题映射`

目标：

- 把市场进入、品类选择、试点、放大、退出放在同一条树状路径里
- 明确每个节点的核心问题、量化指标和 owner

建议回答：

- 当前项目位于哪个 Gate
- 下一个 Gate 需要满足哪些条件
- 每个 Gate 失败后是 Hold、Recycle 还是 Kill

### `0.2 数据置信度与版本控制`

目标：

- 统一证据等级
- 明确版本号、状态、更新时间与来源归档规则

建议回答：

- 哪些数据属于 A/B/C/D 级
- 哪些结论允许使用 B 级支撑，哪些必须 A 级
- 当前版本是否仍然有效

### `0.3 策略假设登记与验证机制`

目标：

- 把所有关键假设显式登记
- 让假设进入“提出—验证—证伪—更新”的工作流

建议回答：

- 这份研究当前依赖哪些假设
- 哪些已经验证，哪些仍未验证
- 哪些假设超期未处理

### `0.4 决策门槛与 Gate 管理`

目标：

- 给 Gate1–Gate5 设定统一阈值
- 固化 Go / Hold / Recycle / Kill 决策规则

建议回答：

- 进入门、立项门、试点门、放大量产门、退出门分别是什么
- 哪些指标是硬门槛
- 哪些指标不达标时必须暂停

### `0.5 责任链与签字流`

目标：

- 建立 RACI 风格的责任与审批链
- 明确谁提议、谁审核、谁批准、谁可否决

建议回答：

- 每个 Gate 的签字顺序是什么
- 是否有 veto 角色
- 是否所有 Gate 都具备最小签字步数

### `0.6 更新周期与失效规则`

目标：

- 定义哪些分析必须月更、季更、事件触发更新
- 明确何种情况下旧结论自动失效

建议回答：

- 各类研究对象的更新频率是什么
- 失效判定是时间失效、事件失效还是偏差失效
- 更新 SLA 由谁负责

### `0.7 字段字典、命名复用与审计要求`

目标：

- 固化字段命名、定义、类型和来源表
- 让跨团队协作不再因为口径漂移而返工

建议回答：

- 哪些字段是跨项目复用字段
- 是否已统一命名风格
- 是否每个关键字段都能追到来源表

## 四、Part 0 的核心判断框架

建议固定成三个一级判断：

### 4.1 这套系统是否可决策

核心看：

- 决策树是否完整
- Gate 是否齐全
- 决策门槛是否量化

### 4.2 这套系统是否可审计

核心看：

- 数据来源是否有证据等级
- 文档是否有版本控制
- 责任链与签字链是否齐全

### 4.3 这套系统是否可复用

核心看：

- 字段字典是否完善
- 更新与失效规则是否明确
- 假设登记是否能跨项目持续复用

## 五、Part 0 建议固定输出的交付件

- 决策树总览页
- 数据置信度分级表
- 假设登记表
- Gate1–5 阈值表
- RACI / 签字链表
- 更新频率与失效规则表
- 字段字典
- 治理审计摘要页

## 六、Part 0 对后续章节的约束作用

### 对 Part 1

- 约束市场规模、需求趋势与渠道判断所允许使用的证据等级

### 对 Part 2

- 约束评论、交易、价格与商品画像字段命名和来源口径

### 对 Part 3

- 约束 RFQ、报价、合规、物流与 landed cost 的审计要求

### 对 Part 4

- 约束渠道进入建议必须通过哪些 Gate

### 对 Part 5

- 约束扩张、回滚、止损、预算分配和实验结论的签字链与失效规则

## 七、这一版初稿的定位

这一版不是最终治理手册，而是 Part 0 的研发起点。

它的目标是先把治理层从“理念”变成“可量化、可开发、可接进现有 report contract”的结构。

下一步应直接进入量化结构与代码骨架，而不是停留在原则说明。
